Dragon Snake - A take on the classic game Snake.
I developed this game in Godot, as a way to learn the engine.
I was already familiar with game development, but most of my experience has been with minimalist frameworks which focus on simplicity and don't have a lot of features.
I think one of my favorite things about the transition from a framework to an engine, is the simplicity of creating UI.
In particular in Godot, I really enjoyed the concept of separating everything into "nodes", and the language GDScript is very enjoyable to program with; it is like a mix of Python and JavaScript.
The game is published on itch.io, and the source code is available on GitHub.

As part of a university project, I developed a virtual computer from scratch, starting from a NAND gate.
The project is called "Nand2Tetris" and is based on a book by the same name.
The computer is called "Hack" and is a 16-bit computer.
The project is divided into two parts: the first part is building the computer's hardware, and the second part is building the computer's software, including developing everything needed to run that software
This involves developing a binary language the computer can understand, and then developing an assembler to convert assembly code into binary code.
Afterwards, I developed a virtual machine to run the binary code, and then a high-level language (called Jack) to write programs in.

This is a gif of the first game I felt comfortable enough recording and sharing to the world. I made this when I was 16 years old in Monogame, an open-source offshoot of Microsoft's XNA framework, which offered a minimalist way to create games without dealing with the lower-level stuff like figuring out how to draw to the screen or load files.
Famous games, such as Terraria and Stardew Valley, were created with this framework. Due to its simplicity and as an avid fan of both since before they were available to the public, this was the framework I ultimately chose to first learn game development with. Engines such as Unity felt too overwhelming at the time.
In hindsight this game wasn't very good; However, I was proud of this at the time, especially the "screen loop" feature I made which gave the feeling of a seamless world. The player could loop from the top of the screen to the bottom seamlessly, giving a fun experience when coupled with the simplistic enemy AI.
